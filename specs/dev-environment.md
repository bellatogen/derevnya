# Спецификация: среда разработки «Пятиречье»

## Цель

Единое рабочее пространство команды и Claude. Git-гигиена закладывается сейчас,
пока в истории один коммит и переписывание истории безболезненно.

## Требования

- **R1.** Корневой `.gitignore` для UE5: `Village/Saved/`, `Village/Intermediate/`,
  `Village/DerivedDataCache/`, `Village/Binaries/`, `Village/Build/`, `.vs/`, `*.sln`,
  `.idea/`, `*.VC.db`, `.DS_Store`, `Thumbs.db`.
- **R2.** Корневой `.gitattributes`: LFS для `*.uasset`, `*.umap`, `*.upack`, `*.fbx`,
  `*.png`, `*.tga`, `*.psd`, `*.wav`, `*.mp3`, `*.mp4`, `*.exr`, `*.hdr`, `*.dll`,
  `*.dmp`, `*.bin`; для текстовых (`*.md`, `*.json`, `*.ini`, `*.uproject`) — `text=auto`.
- **R3.** Переписанная история: `Village/Saved/` и `Village/Intermediate/` отсутствуют
  во всей истории; все бинарники — в LFS; содержимое `Village/Config`, `Village/Content`,
  `Village.uproject` бит-в-бит совпадает с исходным коммитом.
- **R4.** Структура: `docs/`, `specs/`, `narrative/characters/`, `narrative/dialogues/`,
  `narrative/quests/`, `art/` (в пустых папках — `.gitkeep` или короткий README о назначении).
  `Village/` не переименовывается и не переносится.
- **R5.** `CLAUDE.md` в корне — правила:
  - Костя/Коля → `main` напрямую; Claude → только `feature/*` + PR; мержит Павел;
  - Claude никогда не редактирует `.uasset`/`.umap` и вообще файлы в `Village/Content/` —
    бинарный контент только домен Кости; домен Claude — текст, данные, конфиги, доки;
  - один цикл = одна фича = одна ветка, ничего постороннего в диффе; спека до сборки,
    независимое ревью после;
  - force push в `main` запрещён (исключение — разовая операция настройки среды, уже выполнена);
  - конвенция для людей: перед правкой общей карты/ассета — объявить в чате команды
    (git не мержит бинарники);
  - пин движка UE 5.3; смена версии — только решением Кости с записью в README;
  - файлы >2 ГБ в репо не попадают (лимит GitHub LFS); исходники арта >200 МБ —
    в облако, в репо только экспорт;
  - рамки скоупа вертикального среза (из концепта, раздел 32): не строить процедурную
    деревню, сотни NPC, экономику, большой открытый мир — Claude напоминает об этом,
    если задача выходит за рамки.
- **R6.** `README.md`: название и жанр проекта, команда с ролями, требования
  (UE 5.3, git + git-lfs), quickstart (`git lfs install` до клона, как открыть проект),
  правило «Saved/Intermediate не коммитить — они в gitignore», ссылка на CLAUDE.md и `docs/`.
- **R7.** Универсальный каркас формата диалогов: `narrative/dialogues/FORMAT.md` +
  валидный пример `narrative/dialogues/example.json`. Минимальные поля:
  `npc_id`; массив `lines` с `line_id`, `speaker`, `text`,
  `conditions` (свободный key-value объект — словарь условий согласуем позже,
  когда появится скелет сюжета), `responses` (массив `{text, goto}`), опционально
  `notes` для ремарок сценариста. В шапке FORMAT.md статус: «универсальный черновик;
  словарь условий и интеграция с UE DataTable согласуются с Костей при появлении
  скелета сюжета».
- **R8.** `docs/concept.md` — концепт-документ игры. Если в репо или рядом есть
  исходный docx «Идея игры» — конвертируй без потерь смысла; если исходника нет,
  создай `docs/README.md` с пометкой «сюда переносится концепт-документ» и сообщи
  Павлу, что docx нужно приложить.
- **R9.** Итог доставляется прямым пушем (после СТОПа №2), PR для этого цикла не
  нужен — переписанный `main` и есть результат. Файловые коммиты поверх миграции —
  осмысленными сообщениями на русском.

## Ограничения

- Содержимое `Village/Content/**`, `Village/Config/**`, `Village.uproject` по смыслу
  не меняется вообще — цикл инфраструктурный.
- Никаких хуков, обязательных для людей: дисциплина Claude — через CLAUDE.md.
- До СТОПа №1 не выполнять ни одной необратимой операции.

## Краевые случаи

- **E1.** У Кости незапушенные изменения → СТОП №1 не пройден, работа не продолжается.
- **E2.** `git lfs migrate` и `git filter-repo` конфликтуют по порядку → сначала
  filter-repo (удаление путей), затем lfs migrate; после каждого шага — проверка
  целостности `git fsck` и сверка контрольного файла (`Village.uproject` и любые
  3 `.uasset` по хешам с бэкап-веткой).
- **E3.** `example.json` обязан проходить `python3 -m json.tool` и содержать каждый
  элемент, описанный в FORMAT.md.
- **E4.** Пуш LFS отваливается по квоте → доклад Павлу с точной ошибкой, не ретраить в цикле.
- **E5.** `git filter-repo` не установлен → установить (`pip install git-filter-repo`);
  если среда не позволяет — согласованная с Павлом альтернатива, но НЕ `filter-branch` вслепую.

## Определение «готово»

- **D1.** `git log --all --oneline -- Village/Saved Village/Intermediate` — пусто.
- **D2.** `git lfs ls-files | wc -l` покрывает все бинарники; `git rev-list --objects --all`
  не содержит blob'ов >1 МБ вне LFS.
- **D3.** Размер `.git` без LFS-стора (`git count-objects -vH`) — единицы мегабайт.
- **D4.** Хеши `Village.uproject` и трёх контрольных `.uasset` (после LFS-checkout)
  совпадают с бэкап-веткой — контент не пострадал.
- **D5.** Все файлы R1–R8 существуют и содержательны (не заглушки), `example.json` валиден.
- **D6.** Свежий клон во временную папку (`git clone --local` + `git lfs checkout`)
  открываемо-целостен: `.uproject` на месте, Content подтянулся из LFS.
