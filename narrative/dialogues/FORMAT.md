# Формат диалогов

> Статус: универсальный черновик. Словарь условий (`conditions`) и интеграция с UE
> DataTable согласуются с Костей при появлении скелета сюжета — сейчас формат намеренно
> не привязан к конкретным Gameplay Tags.

Каждый диалог — JSON-файл в `narrative/dialogues/`.

## Структура

```json
{
  "npc_id": "string",
  "lines": [
    {
      "line_id": "string",
      "speaker": "string",
      "text": "string",
      "conditions": {},
      "responses": [
        { "text": "string", "goto": "string" }
      ],
      "notes": "string (опционально)"
    }
  ]
}
```

## Поля

- **`npc_id`** — идентификатор NPC, к которому привязан диалог.
- **`lines`** — массив реплик диалога.
  - **`line_id`** — уникальный в пределах диалога идентификатор реплики (для `goto`).
  - **`speaker`** — кто говорит (имя NPC или `"player"`).
  - **`text`** — текст реплики.
  - **`conditions`** — свободный key-value объект условий показа реплики. Словарь
    ключей ещё не согласован — заполняется по мере появления скелета сюжета.
  - **`responses`** — варианты ответа игрока, каждый с `text` (текст варианта) и
    `goto` (`line_id` следующей реплики).
  - **`notes`** — необязательное поле для ремарок сценариста (тон, режиссура сцены,
    заметки для локализации). В геймплей не попадает.

См. `example.json` — валидный минимальный пример.
