# CLAUDE.md

Правила работы Claude Code в репозитории проекта «Пятиречье».

## Команда и роли

- **Костя** — гейм-дизайн, UE5, весь бинарный контент. Коммитит в `main` напрямую.
- **Коля** — концепт-арт. Коммитит в `main` напрямую.
- **Павел** — нарратив, диалоги; оператор Claude и единственный, кто мержит PR.
- **Claude** — только ветки `feature/*` + Pull Request. Никогда не коммитит и не пушит
  напрямую в `main`.

## Границы домена Claude

- Claude **никогда** не редактирует `.uasset`/`.umap` и вообще любые файлы внутри
  `Village/Content/` — бинарный контент строго домен Кости.
- Домен Claude: текст, данные, конфиги, документация (`docs/`, `specs/`, `narrative/`,
  корневые `.md`-файлы).
- Перед правкой общей карты/ассета в UE — участник объявляет об этом в чате команды
  (git не мержит бинарники, конфликт разрешить некому).

## Процесс

- Один цикл работы = одна фича = одна ветка `feature/*`, ничего постороннего в диффе.
- Спецификация — до сборки (`specs/`); независимое ревью — после.
- Мержит PR только Павел.

## Git-гигиена

- Force push в `main` **запрещён**. Единственное исключение — разовая операция
  первичной настройки среды (git-lfs миграция, переписывание истории), уже выполненная;
  повторений не будет.

## Движок

- Пин версии: **Unreal Engine 5.3**.
- Смена версии движка — только решением Кости, с записью здесь, в CLAUDE.md.

## Ограничения репозитория

- Файлы **>2 ГБ** в репозиторий не попадают (лимит GitHub LFS).
- Исходники арта (PSD-слои, рабочие файлы) **>200 МБ** — хранятся в облаке, в репозиторий
  попадает только финальный экспорт (см. `art/README.md`).

## Рамки скоупа (вертикальный срез)

Согласно концепту (см. `docs/concept.md`, раздел 32) — не строить в рамках вертикального
среза:
- процедурную деревню;
- сотни NPC;
- экономику;
- большой открытый мир.

Claude напоминает об этом, если задача выходит за рамки вертикального среза.
